## Git Flow

2010년 Vincent Dressen이 제시한 깃 브랜치 전략이다. 주로 중간 규모 이상의 개발팀에서 널리 사용되는데, 방법론은 아래와 같다.

![https://nvie.com/img/git-model@2x.png](https://nvie.com/img/git-model@2x.png)

## Git Flow 방법론

1. **master**: 제품으로 출시될 수 있는 브랜치로, 항상 안정적인 상태를 유지한다.
2. **develop**: 다음 출시 버전을 개발하는 브랜치로, 항상 최신 개발 상태를 유지한다.
3. **featrue**: 다음 출시 버전을 개발하는 브랜치로, develop 브랜치에서 분기해 개발한다. 작업이 완려되면 develop 브랜치에 병합한다.
4. **release**: 다음 버전 출시를 위한 브랜치로, develop 브랜치에서 분기해 출시 준비 작업을 진행한다. QA, 테스트 등을 거쳐 안정성 확보 후 master 브랜치와 develop 브랜치로 병합한다.
5. hotfix: 출시된 제품에서 발생한 버그 등 긴급한 수정 작업을 위한 브랜치로, master 브랜치에서 분기하여 수정 작업을 진행한다. 작업이 완료되면 master 브랜치와 develop 브랜치에 병합한다.

위 방법으로 브랜치 전략을 사용하면 개발 과정에서 버그를 최소화하고 안정적인 제품 출시를 보장할 수 있다.

여러 개발자들이 함께 작업하는 경우 각자의 작업을 분리하여 개발할 수 있어 효율적인 협업이 가능하다.
